  1. 创建一个源项目仓库（源代码专用，不用于部署）
    比如新建一个 GitHub 仓库叫：

复制
编辑
hexo-blog-source
然后把你本地的 Hexo 源代码（包括 package.json, _config.yml, themes/, source/ 等）推送上去：

bash
复制
编辑
cd 你的Hexo项目目录
git init
git remote add origin https://github.com/你的用户名/hexo-blog-source.git
git add .
git commit -m “init blog source”
git push -u origin main
✅ 2. 将 hexo d 继续部署到 shenggeliashui.github.io
也就是说：

hexo-blog-source：多人协作 + 撰写文章 + 开发主题 + 执行 hexo d

shenggeliashui.github.io：只用来托管生成后的静态文件（不要手动修改）

✅ 3. 协作者如何参与
别人只需要 clone 你的 hexo-blog-source 项目，然后本地执行：

bash
复制
编辑
npm install
hexo s
就可以本地开发、写文章、改样式，然后：

bash
复制
编辑
hexo d # 部署到你的 .github.io 仓库
前提是他们有部署权限，或者通过 PR 提交你来部署。
